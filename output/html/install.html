Wie man Samba bekommt und installiert

Binär Pakete von Samba gibt es mittlerweile in fast jeder Linux oder Unix Distribution. Auch gibt es einige Pakete auf der Samba Homepage. Lesen Sie bitte in der Bedienungsanleitung ihres Betriebsystems nach wie Sie auf ihrem System Samba installieren können.

Wenn Sie Samba von den Quelldateien selbst installieren möchten, bekommen Sie hier weitere Informationen.

Samba konfigurieren (smb.conf)

Die Samba konfiguration wird in der smb.conf vorgenommen die Sie üblicherweise in /etc/samba/smb.conf oder /usr/local/samba/lib/smb.conf finden. Sie können diese Datei selbst editieren oder eines der vielen grafischen Oberflächen wie das Webbasierte SWAT nutzen, welches mit Samba mitgeliefert wird.

Die Syntax der Konfiguration

Die smb.conf nutzt nahezu die gleiche Syntax wie die verschiedenen .ini Dateien von Windows 3.1: Jede Datei enthält verschiedene Sektionen wobei jede dieser Sektionen durch Klammern ([]) getrennt sind. Jede Sektion besteht aus keinem oder mehreren Schlüssel/Wert paaren die durch das Gleichheitszeichen (=) miteinander verbunden werden. Dies ist eine Klartext ASCII Datei, die Sie mit ihrem Lieblings Editor bearbeiten können.

Jede Sektion der smb.conf Datei repräsentiert eine Freigabe auf dem Samba Server bis auf die “global”, dies ist eine spezielle Sektion da sie Einstellungen enthält die den gesamten Samba Server beeinflussen und nicht nur einzelne Freigaben.

??? enthält eine sehr kleine smb.conf.

Example 2.1. Eine minimale smb.conf

[global]
workgroup = WKG
netbios name = MYNAME
[share1]
path = /tmp
[share2]
path = /my_shared_folder
comment = Some random files

Samba starten

Samba an sich besteht aus zwei oder drei sog. daemons. Ein deamon ist eine Unix Applikation die im Hintergrund läuft und Dienste zur Verfügung stellt. Ein Beispiel eines solchen Dienstes ist der Apache Web Server für den der daemon httpd heißt. Im Falle von Samba gibt es drei daemons von denen mindestens zwei benötigt werden.

Der Samba Server besteht aus folgenden daemons:

nmbd

Dieser daemon behandelt alle Namenregistrierungen und Namensauflösungs anfragen. Es ist das Primäre Werkzeug für das schmöckern eines Netzwerkes. Er ist in der lage mit jedem UDP basierten Protokoll umzugehen. Das Kommando nmbd sollte der erste daemon sein der im Rahmen des Samba Startvorgangs geladen wird.

smbd

Dieser daemon behandelt alle TCP/IP basierenden Verbindungen für Datei- und Druckbasierten operationen. Ebenfalls handelt dieser daemon mit der Benutzerauthentisierung und sollte direkt nach dem Start von nmbd aufgerufen werden.

winbindd

Dieser daemon sollte gestartet werden, wenn der Samba Server ein Mitglied einer Windows NT4 oder ADS (ActiveDirectoryService) Domäne ist. Er ist ebenfalls nötig, wenn Samba in einem Vertrauenverhältnis zu anderen Domänen steht. Der winbindd daemon überprüft die smb.conf auf das vorhanden sein der idmap uid und idmap gid Parameter. Sollten Sie fehlen gibt der winbindd einen Fehler aus und startet nicht.

Wenn Samba als Paket einer Distribution installiert wird liegt meist ein Start-Stop Script für den Ordnungsgemäßen start von Samba vor ist aber eher ein positives Merkmal der Distribution. Bitte konsultieren Sie das Administrationshandbuch ihres Betriebsystems bezüglich des einbinden eines solchen Start-Stop Scripts.

Es gibt Beispiel Konfigurationsdateien in einem separaten Unterverzeichnis der Distribution. Es lohnt sich die vorgegebene smb.conf.default als Grundlage der persönlichen Konfiguration zu verwenden da sie viele Kommentare zu den verschiedenen Optionen enthält.

Example 2.2. Eine weitere vereinfachte smb.conf Datei

[global]
workgroup = MIDEARTH
[homes]
guest ok = no
read only = no

Diese Konfiguration erlaubt es jedem der einen Benutzeraccount auf dem Server hat sich mit seinem Login Namen auf die Freigabe homes zu verbinden. (Hinweis: Der name der Arbeitsgruppe/Domäne sollte hier so wie gewünscht gesetzt werden, der Standard für diese Option ist WORKGROUP.)

Stellen Sie sicher, dass sie die smb.conf Datei im richtigen Verzeichniss gespeichert haben.

Test der Konfiguration mit testparm

Es ist wichtig ihrer Konfigurationsänderung mit testparm zu überprüfen, wenn ihre Konfiguration richtig ist sollte testparm eine Liste der Verfügbaren Services auflisten, andernfalls wird es eine Fehlermeldung ausgeben. Überprüfen Sie ihre Konfiguration mittels: 

	root#  testparm /etc/samba/smb.conf

Testparm wird ihre Konfiguration überprüfen und mögliche unbekannte Parameter oder fehler in der Syntax ausgeben.

Es ist sehr empfehlenswert testparm nach einer Änderung der smb.conf auszuführen.

SWAT

SWAT ist eine Weboberfläche die dazu genutzt werden kann die Konfiguration von Samba zu erleichtern. Es kann möglich sein, dass SWAT nicht in der Paketinstallation ihrer Distribution enthalten ist, dann finden Sie es aber meist in einem separaten Paket. Bitte beachten Sie die SWAT man-page wenn Sie SWAT aus den Quelldateien kompilieren, installieren und konfigurien wollen.

Um SWAT zu starten verwenden Sie einfach ihren favorisierten Web-Browser und verwenden folgende URL http://localhost:901/. Ersetzen Sie localhost mit dem Namen des Computers auf dem Sie Samba installiert haben wenn Sie den Web-Browser von einem anderen PC als den Samba Server starten.

SWAT kann von jedem Browser eines IP-Netzes aus genutzt werden aber beachten Sie dass bei Verbindungen von einem entfernten Rechner die Gefahr besteht dass ihre Passwörter abgehört werden können, da diese in Klartext über das Kabel transportiert werden.

Auflistung von Freigaben auf dem Server

Verwenden Sie folgendes Kommando um die verfügbaren ihres Samba Server aufzulisten: 

$ smbclient -L yourhostname

Sie sollten nun eine Liste der verfügbaren Freigaben ihres Servers sehen, wenn nicht dann ist etwas nicht richtig konfiguriert. Dieses Kommando können Sie ebenfalls nutzen um die Freigaben anderer Server z.B. Windows 2000 zu betrachten.

Wenn Sie user-level Sicherheit als Sicherheit für ihren Server gewählt haben, sollten Sie nach dem Kommando nach einem Passwort für ihren Benutzer gefragt werden bevor Sie die Auflistung sehen. Lesen Sie die smbclient man-page für weitere Informationen. Sie können das Kommando mittels der Option -N in der Kommandozeile dazu bringen sich auch ohne Passwort die Freigabeliste anzuzeigen.

Verbindung zu einem UNIX Client aufbauen

Verwenden Sie folgendes Kommando: 

$ smbclient  //yourhostname/aservice

Normalerweise ist yourhostname der Name des Hosts auf dem sich der smbd befindet und aservice ist ein belieber Service den dieser Host zur Verfügung stellt. Versuchen Sie es mit ihrem Benutzernamen, wenn Sie eine Sektion [homes] in der smb.conf haben.

Beispiel: Wenn der UNIX Host bambi heißt und ein gültiger Login Name fred ist könnten Sie folgendes schreiben: 

$ smbclient //bambi/fred

Verbindung von einem entfernten SMB Client

Nun da Samba lokal richtig funktioniert können Sie versuchen sich von einem anderen Rechner aus an ihrem Samba Server anzumelden. Innerhalb weniger minuten sollte der Samba Server in ihrer Netzwerkumgebung ihres Windows Subnetzes sichtbar sein. Versuchen Sie ihren Samba Server dort zu finden oder versuchen Sie eine Netzwerkverbindung mit ihrem Server herzustellen.

Um eine Netzwerkverbindung unde DOS, Windows oder OS/2 Rechnern herzustellen führen Sie folgendes Kommando aus: 

	C:\> net use d: \\servername\service

Versuchen Sie zu drucken:

C:\> net use lpt1:	\\servername\spoolservice


C:\> print filename

Verbreitete Fehler

Folgende Fragen und Themen tauchen immer wieder in der Samba Mailing Liste auf.

Große Anzahl von smbd Prozessen

Samba enthält drei Core Programme: nmbd ist der Namensserver daemon, smbd ist der Server Message daemon und winbindd ist der daemon der mit Domänen Controllern arbeitet.

Wenn Samba nicht als WINS Server fungiert sollte es nur eine Instanz des nmbd daemons auf ihrem System geben. Andernfalls sollten zwei nmbd Prozesse laufen einer von den beiden behandelt die WINS Anfragen.

smbd behandelt alle Verbindungsanfragen. Er ruft sich selbst für jede neue Client-Verbindung auf. Das ist der Grund warum Sie mehrere Prozesse von smbd sehen können, einen für jede Client-Verbindung.

winbindd läuft in einem oder zwei Prozessen, abhängig davon ob er im split mode läuft (in diesem Fall hat er zwei Instanzen).

Fehler Meldung: open_oplock_ipc

Eine Fehlermeldung wird in der Log Datei beim starten von smbd angezeigt: “open_oplock_ipc: Failed to get local UDP socket for address 100007f. Error was Cannot assign requested.”

Ihr Loopback Interface funktioniert nicht korrekt. Vergewissern Sie sich dass es korrekt konfiguriert wurde. Das Loopack Interface ist eine interne (virtuelles) Netzwerkkarte mit der IP-Adresse 127.0.0.1. Lesen Sie in ihrer Betriebssystem Dokumentation nach, wie Sie ihr Loopback interface einrichten.

“The network name cannot be found”

Dieser fehler kann von folgenden Mißkonfigurationen stammen:

  • Sie haben einen nicht existierenden Pfad für eine Freigabe in der smb.conf angegeben.

  • Der Benutzer mit dem Sie versuchen auf eine Freigabe zu zugreifen hat nicht die erforderlichen Rechte um auf den UNIX-Pfad zu zugreifen. Sowohl das Leserecht (r) sowie das Zugriffsrecht (x) sollten dem Benutzer erlaubt sein.

  • Die Freigabe auf die Sie versuchen zu zugreifen existiert nicht.
