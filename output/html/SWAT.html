Es gibt viele und verschiedene Meinungen bezüglich des Nutzens von SWAT. Gleichgültig wieviel Mühe man sich gibt ein perfektes Konfigurationswerkzeug zu erstellen, ist die Benutzung eine Frage des persönlichen Geschmacks. SWAT ist ein Werkzeug das eine Web-basierte Konfiguration von Samba ermöglicht. Es beinhaltet einen Wizard zur schnellen Konfiguration von Samba, besitzt eine kontextabhängige Hilfe für jeden smb.conf Parameter, überwacht den aktuellen Zustand der Verbindung und erlaubt eine netzwerkweite MS Windows Passwortverwaltung.

Features und Nutzen

SWAT ist Teil der Samba Suite. Das Hauptprogramm heißt swat und wird über xinetd gestartet (siehe ??? fuer Einzelheiten).

SWAT verwendet Samba-eigene Komponenten, um die von der jeweiligen Samba-Version unterstützten Parameter zu lokalisieren. Anders als externe Werkzeuge und Hilfsmittel bleibt SWAT bei Parameteränderungen auf dem aktuellen Stand. SWAT bietet kontextabhaägige Hilfe fuer jeden Konfigurationsparameter direkt aus Man Page Einträgen heraus.

Manche Netzwerkadministratoren schreiben Dokumentationen in Form von Kommentaren innerhalb von Konfigurationsdateien. Für diese wird SWAT ein eher unbeliebtes Werkzeug sein. SWAT speichert die Konfigurationsdatei in keine Mischform, sondern speichert ausschliesslich Parameterwerte. Wenn SWAT also die smb.conf schreibt, schreibt es nur jene Parameter, die von den Standardwerten abweichen. Das bedeutet, dass Kommentare und nicht unterstützte Parameter nicht mehr in der smb.conf erscheinen werden. Hinzu werden die Parameter in ein interne Ordnung geschrieben.

Note

Vor dem Benutzen von SWAT SWAT ersetzt eine bereits im System vorhandene smb.conf mit einer von SWAT optimierten Version, die von allen vorher eingefügten Kommentaren und Nicht-Standardwerten bereinigt ist.

Grundlagen und technische Hilfen

Dieser Abschnitt erklärt wie man SWAT aktiviert, wie man es sicherer macht und wie man Probleme bei der Mehrsprachenunterstützung löst.

Überprüfen der SWAT Installation

Bevor man einen Rechner beginnt für SWAT zu konfigurieren, ist zu überprüfen, ob es überhaupt installiert ist. Dies mag einigen trivial erscheinen, dennoch installieren einige Distributionen SWAT nicht standardmäßig, obwohl es auf den Medien des Distributors vorhanden ist.

Ist man sich sicher, dass SWAT installiert ist, ist es notwendig zu überprüfen, dass die Installation das swat Binary wie auch alle unterstützenden Text- und Webseiten enthält. Bei einer Reihe von Linux-Distributoren fehlten diese vollständig, obwohl das swat Binary installiert war.

Nachdem man sich sicher ist, dass wirklich alles installiert ist, stellt man sicher, dass SWAT in der Kontrolldatei von inetd oder xinetd aktiviert ist, je nachdem, welches von beiden vom Betriebsystem verwendet wird.

Lokalisieren der swat Datei

Um sicher zu stellen, dass SWAT installiert ist, lokalisiert man das swat Binary im System. Es könnte in folgenden Verzeichnissen liegen:

/usr/local/samba/bin der standardmäßige swat Pfad.
/usr/sbin der Standardort vieler Linux-Systeme.
/opt/samba/bin

Der aktuelle Ort ist abhängig von der Wahl des Distributors oder des vom Administrator gewählten Installationspfades.

Es gibt eine Anzahl verschiedener Methoden den Ort der Installation des swat Binaries festzustellen. Die folgenden Methoden können hilfreich sein:

Wenn swat sich im aktuellen Suchpfad des Betriebssystems befindet, ist es einfach zu finden. Man kann sich die Kommandoparameter von swat wie folgt anzeigen lassen: 

frodo:~ # swat -?
Usage: swat [OPTION...]
  -a, --disable-authentication         Authentifizierung abschalten (demo mode)

Help options:
  -?, --help                           Zeigt diese Hilfe an
  --usage                              Zeigt eine kurze Benutzerhilfe an

Common samba options:
  -d, --debuglevel=DEBUGLEVEL          Setzt debug level
  -s, --configfile=CONFIGFILE          Benutze alternative Konfigurationsdatei
  -l, --log-basename=LOGFILEBASE       Basisname für log/debug Dateien
  -V, --version                        Gibt die Version aus

Lokalisieren der SWAT Hilfedateien

Da man nun herausgefunden hat, dass swat im Suchpfad ist, ist es einfach, es im System zu identifizieren. Hier ist ein anderer einfacher Weg dies zu bewerkstelligen: 

frodo:~ # whereis swat
swat: /usr/sbin/swat /usr/share/man/man8/swat.8.gz

Wenn dieser Versuch das swat Binary zu finden mislingt, ist ein weiterer Versuch notwendig: 

frodo:/ # find / -name swat -print
/etc/xinetd.d/swat
/usr/sbin/swat
/usr/share/samba/swat
frodo:/ #

Die Ausgabe zeigt, dass sich eine Kontrolldatei für xinetd auf der Maschine vorhanden ist. Der Ort der SWAT Binarydatei ist /usr/sbin/swat und die Hilfedateien befinden sich im Verzeichnis /usr/share/samba/swat.

Folgend muss überprüft werden, wo swat annimmt das seine Hilfedateien liegen. Dies kann wie folgt geschehen: 

frodo:/ # strings /usr/sbin/swat | grep "/swat"
/swat/
...
/usr/share/samba/swat
frodo:/ #

Der /usr/share/samba/swat/ Eintrag ist der Ort, in dem sich die von SWAT benötigten Dateien befinden. Um sicherzustellen, dass alle Dateien in diesem Verzeichnis vorhanden sind, gibt man folgendes ein:

SWAT aktivieren

SWAT sollte installiert werden, um es via inetd oder xinetd starten zu können. Abhängig vom UNIX/Linux-System ist dies entweder inetd oder xinetd.

Die Handhabung und der Ort variiert hierbei vom verwendeten Betriebssystem. Die Kontrolldatei (oder Dateien) findet man im Allg. unter /etc/inetd.conf oder (für xinetd) im Verzeichnis /etc/[x]inet[d].d.

Der Eintrag für den älteren Typ der Kontrolldatei kann sein: 

	# swat is the Samba Web Administration Tool
	swat stream tcp nowait.400 root /usr/sbin/swat swat

Ein Eintrag für die neuere Variante der Kontrolldatei in xinetd könnte sein: 

# default: off
# description: SWAT is the Samba Web Admin Tool. Use swat \
#              to configure your Samba server. To use SWAT, \
#              connect to port 901 with your favorite web browser.
service swat
{
	port    = 901
	socket_type     = stream
	wait    = no
	only_from = localhost
	user    = root
	server  = /usr/sbin/swat
	log_on_failure  += USERID
	disable = yes
}

Beide der oben angegebenen Beispiele bedingen, dass swat im /usr/sbin Verzeichnis liegt. Zusätzlich benötigt SWAT ein Verzeichnis in dem es die Hilfedateien wie auch die Kontrollinformationen findet. Der Standardpfad für diese auf den meisten Linux-Systemen ist /usr/share/samba/swat. Der Pfad für die Samba-Standards ist /usr/local/samba/swat.

Bei einem Zugriff auf SWAT wird nach einer Anmeldung gefragt. Wenn man sich in SWAT als Nicht-Root-Benutzer anmeldet, sind die einzigen Berechtigungen, einige bestimmte Konfigurationsoptionen nur zu lesen sowie der Zugriff auf die Passwortverwaltung. Die für Nicht-Root-Benutzer angezeigten Buttons sind: HOME, STATUS, VIEW, PASSWORD. Die einzige Seite, die Änderungen erlaubt, ist PASSWORD.

Meldet man sich als Benutzer root an, erhält man volle Kontrolle über sämtliche Funktionen in SWAT. Die angezeigten Buttons sind: HOME, GLOBALS, SHARES, PRINTERS, WIZARD, STATUS, VIEW, PASSWORD.

Absichern von SWAT mit SSL

Viele Leute haben angefragt, wie man SWAT mit SSL aufsetzt, um so eine sichere Fernadministration zu erlauben. Hier ist eine funktionierende Methode, zur Verfügung gestellt von Markus Krieger.

Die Modifikationen der SWAT Installation wie folgt:

  1. Installieren von OpenSSL.

  2. Generieren eines Zertifikats und eines privaten Schlüssels. 

    root# /usr/bin/openssl req -new -x509 -days 365 -nodes -config \
	/usr/share/doc/packages/stunnel/stunnel.cnf \
	-out /etc/stunnel/stunnel.pem -keyout /etc/stunnel/stunnel.pem

  3. Entfernen des swat-Eintrages aus [x]inetd.

  4. Ausführen von stunnel. 

    root# stunnel -p /etc/stunnel/stunnel.pem -d 901 \
	 -l /usr/local/samba/bin/swat swat

Danach einfach mit swat über die URL https://myhost:901 verbinden, akzeptieren des Zertifikats und die SSL-Verbindung steht.

SWAT Mehrsprachenunterstützung aktivieren

SWAT kann so konfiguriert werden, dass es seine Nachrichten in der Sprache anzeigt, die im Webbrowser eingestellt ist. Es wird SWAT im Accept-Language-Header der HTTP-Antwort übergeben.

Um dieses Feature zu aktivieren:

  • Installieren der passenden msg Dateien aus dem Samba source/po Verzeichnis nach $LIBDIR.

  • Setzen des korrekten Werts für display charset.

  • Setzen der Sprache im Browser.

Der Name der msg Datei ist gleich der Sprach-ID vom Browser. Zum Beispiel en für "English", ja für "Japanisch", fr für "Französisch".

Wenn einem einige der Nachrichten nicht gefallen, oder gar keine msg Dateien für die eigene oder präferierte Sprache vorhanden ist, kann man einfach die en.msg Dateien in das Verzeichnis für “Meine Sprache ID.msg” kopieren und passende Nachrichten zu jedem “msgstr” eintragen. Zum Beispiel für it.msg, die msg Datei für Italienisch, trägt man ein: 

msgid "Set Default"
msgstr "Imposta Default"

usw. Wenn man einen Fehler findet oder eine neue msg Datei erstellt, mailt diese bitte, um sie für das nächste Release von Samba bereitzustellen.

Anmerkend sei erwähnt, dass wenn dieses Feature aktiviert wird und display charset stimmt nicht mit der im Browser eingestellten Sprache überein, kann die Ausgabe von SWAT durcheinander geraten. In einer zukünftigen Samba-Version wird SWAT immer UTF-8 kodierte Nachrichten anzeigen. Es wird dann nicht mehr nötig sein den smb.conf Dateiparameter für die Sprache zu setzen.

Übersicht und ein Schnelldurchlauf

SWAT ist ein Werkzeug, das von vielen genutzt wird, um Samba zu konfigurieren oder um auf die nützlichen Links zu wichtigen Inhalten und Referenzmaterial, wie auf den Inhalt dieser Dokumentation, zuzugreifen. Wie auch anderes nützliches, um Probleme mit dem Windows-Netzwerk zu lösen.

Die SWAT Homepage

Die SWAT Titelseite bietet Zugriff auf die aktuellste Samba-Dokumentation. Von dieser Seite können auf die Manual Pages jeder Samba-Komponente zugegriffen werden, wie die Samba HOWTO-Sammlung (dieses Dokument)und auch auf das Buch von O'Reilly “Using Samba.”

Administratoren die ihre Samba-Konfiguration überprüfen möchten, finden nützliche Hilfe in den Man Pages der einzelnen Diagnose-Werkzeuge, die auch auf der SWAT Homepage im Internet verfügbar sind. Ein nicht erwähntes, aber teils nützliches Diagnose-Werkzeug findet man hier ethereal.

Warning

SWAT kann so konfiguriert werden, dass es in einem Demo Modus läuft. Dieser Modus ist nicht empfohlen, da es SWAT ohne Authentifizierung, aber mit sämtlichen administrativen Möglichkeiten ausführt. Es erlaubt Änderungen an der smb.conf, wie auch alle anderen Tätigkeiten mit root-Rechten. Die Option, die dies erlaubt, ist das -a Flag für swat. Dies sollte man nicht in einer Produktionsumgebung verwenden!

Globale Einstellungen

Der GLOBALS Button öffnet die Seite, auf der es möglich ist, die globalen Parameter in der smb.conf zu konfigurieren. Es gibt zwei Arten, wie diese Parameter angezeigt werden:

  • Basic Zeigt allgemeine Konfigurationsoptionen an.

  • Advanced Zeigt Konfigurationsoptionen an, die in komplexeren Umgebungen gebraucht werden.

Um die Optionen im erweiterten Modus zu editieren, klickt man auf Advanced, im anderen Fall auf Basic. Dies kann auch erreicht werden, wenn auf den Radio-Button drückt und dann auf den Commit Changes Button.

Nachdem man Änderungen an Konfigurationsparametern vorgenommen hat, sollte man sicher gehen, dass man auf den Commit Changes Button gedrückt hat, bevor man diese Seite verlässt, da sonst alle Änderungen verloren gehen.

Note

SWAT besitzt eine kontextabhängige Hilfe. Um herauszufinden, wofür jeder Parameter steht, welchen Zweck er erfüllt, drückt man auf den Help Link links neben dem Konfigurationsparameters.

Freigabe (Share) Einstellungen

Um eine bereits konfigurierte Freigabe zu bearbeiten, drückt man auf das Pull-Down-Menü zwischen Choose Share und Delete Share und wählt die Freigabe aus, mit der man arbeiten will. Um die Einstellungen zu editieren, wählt man den Choose Share Button. Um die Freigabe zu löschen den Delete Share button.

Um eine neue Freigabe zu erzeugen, befindet sich neben dem mit Create Share bezeichneten Button ein Textfeld, in das man den Namen der Freigabe eingibt. Die Freigabe erzeugt man schließlich mit Create Share.

Druckereinstellungen

Einen bereits eingerichteten Drucker bearbeitet man mit dem Pull-Down-Menü zwischen Choose Printer und Delete Printer, aus dem man den zu bearbeitenden Drucker auswählt. Um die Einstellungen zu bearbeiten benutzt man den Choose Printer Button, um den Drucker aus der Liste zu löschen Delete Printer.

Einen neuen Drucker legt man in dem Textfeld neben dem Button Create Printer an, indem man den Namen des freizugebenden Druckers eingibt. Frei gibt man ihn schließlich durch klicken auf Create Printer.

Der SWAT Wizard

Der Zweck des SWAT Wizards ist es, dem sachkundigen Microsoft Netzwerkadministrator die Konfiguration von Samba so einfach wie möglich zu machen.

Die Wizard-Seite bietet ein Werkzeug, um die smb.conf Datei in einem optimierten Format zurückzuschreiben. Genau dies geschieht auch, wenn man den Commit Button drückt. Der Unterschied zwischen dem Rewrite Button und dem Commit Button ist, dass Rewrite alle Änderungen ignoriert und Commit sämtliche gemachten Änderungen vornimmt.

Der edit Button erlaubt das Editieren der Minimaloptionen, die notwendig sind, einen funktionsfähigen Samba-Server zu konfigurieren.

Schließlich gibt es noch eine begrenzte Anzahl von Optionen, die bestimmen, für was der Samba-Server konfiguriert werden soll: Ob als WINS-Server, anteilig als WINS-Client, oder ob er ohne WINS-Unterstützung arbeiten soll. Mit einem Klick kann man auswählen, ob man die Heimverzeichnisse von Benutzern freigeben will.

Die Status-Seite

Die Status-Seite erfüllt nur einen geringen Nutzen. Es erlaubt die Überwachung der Samba-Daemonen, wobei die Schlüssel-Daemonen, die den Samba-Server ausmachen, smbd, nmbd, winbindd sind.

Die Daemonen können individuell überwacht werden oder als eine zusammenhängende Gruppe. Hinzu kann man eine automatische Aktualisierung der Seite einstellen. Wenn MS Windows Clients mit Samba interagieren, werden neue smbd-Prozesse kontinuierlich eingeplegt. Mit minimalem Aufwand lassen sich über die automatische Aktualisierung sich ändernde Zustände verfolgen.

Zuletzt lässt sich die Status-Seite dazu benutzen, bestimmte smbd-Clientverbindungen zu unterbrechen, um Dateien freizugeben, die u.U. gesperrt sind.

Die Übersichtsseite

Sie ermöglicht es dem Administrator sich die optimierte smb.conf Datei anzusehen und, für die Masochisten unter uns, erlaubt es ebenso die Ansicht aller möglichen globalen Konfigurationsparameter und ihrer Einstellungen.

Die Passwortänderungs-Seite

Die Passwortänderungs-Seite ist ein sehr beliebtes Werkzeug, dass das Erstellen, Löschen, De- und Reaktivieren von MS Windows Benutzern auf der lokalen Maschine erlaubt. Alternativ kann man es zur Änderung von lokalen Passwörtern verwenden.

Ist man als Nicht-Root-Benutzer angemeldet, muss der Benutzer von SWAT bei Änderungen von Passwörter das alte und zweimal das neue Passwort eingeben. Als root wird nur das neue Passwort benötigt.

Eine weitere bekannte Benutzung des Werkzeugs ist es, Benutzerpasswörter über eine Reihe von MS Windows-Servern hinweg zu ändern.

Fehlherhafte Ausgaben der SWAT Übersichtsseite

Wenn sich die display charset und dos charset Parameter unterscheiden, wird die Übersichtsseite fehlerhaft angezeigt. Der display charset Parameter muss dieselbe Kodierung wie der msg-Datei besitzen. Für Japan bedeutet das, dass display charset auf CP932 gesetzt sein muss.

Das Setzen von unix charset = EUCJP-MS erzeugt das Problem.
